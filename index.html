Setup Guide

Four steps to connect Claude Code to your Anypoint Platform organization.

Prerequisites

  • Node.js 18+ — brew install node or download from nodejs.org
  • Claude Code — npm install -g @anthropic-ai/claude-code
  • Anypoint Platform account — with Access Management permissions
  • Java 17 + Maven (only for building/deploying Mule apps) — brew install openjdk@17 maven

Save the Client Secret now. It will not be shown again. If you lose it, you'll need to regenerate credentials.
5

Connect Claude Code

Open your terminal and run this command, replacing the two placeholder values with your credentials:

claude mcp add mulesoft \ --scope user \ -e ANYPOINT_CLIENT_ID=paste_your_client_id \ -e ANYPOINT_CLIENT_SECRET=paste_your_client_secret \ -e ANYPOINT_REGION=PROD_US \ -- npx -y @mulesoft/mcp-server start
Scope options: Use --scope user to make it available in all your projects. Use --scope local to restrict it to the current project only.

EU region? Change PROD_US to PROD_EU.
6

Verify the Connection

Run this command to confirm Claude Code can reach Anypoint:

Terminal
$ claude mcp list
Checking MCP server health...

mulesoft: npx -y @mulesoft/mcp-server start - ✓ Connected

If you see ✓ Connected, you're ready to go. Head to the First 10 Things to Try tab.

Common Errors

Double-check your Client ID and Client Secret for typos. The most common issue is an extra space or missing character when pasting. Re-run claude mcp remove mulesoft and then re-add with the corrected values.
Your Connected App is missing scopes. Go back to Access Management > Connected Apps, edit the app, and add the missing scope group. You don't need to regenerate credentials — just add the scope and save.
Node.js is not installed or not on your PATH. Install it with brew install node (macOS) or download from nodejs.org. After installing, close and reopen your terminal.
Building and deploying Mule projects requires Java 17 and Maven. Install with: brew install openjdk@17 maven. Then set JAVA_HOME: export JAVA_HOME=$(/usr/libexec/java_home -v 17). Add that line to your ~/.zshrc to make it persistent.
If your Anypoint org is in the EU, you must set ANYPOINT_REGION=PROD_EU. Remove and re-add the server: claude mcp remove mulesoft then re-run the add command with PROD_EU.

Best Practices

Get the most out of MuleSoft + Claude Code with these guidelines.

🔒 Security

  • Rotate credentials quarterly. Delete and re-create the Connected App every 90 days, then update with claude mcp remove mulesoft and re-add.
  • Use minimum scopes for production. Start with all scopes during exploration, then tighten to only what you need for production workflows.
  • Use --scope local for shared machines. This restricts the MCP server credentials to the current project directory only.
  • Never commit credentials. The MCP config is stored in ~/.claude/, not in your project repo. Keep it that way.

📦 Exchange-First Development

  • Search before you build. Before writing any integration code, ask Claude to search Exchange. There are thousands of pre-built assets — connectors, templates, accelerators, and API specs.
  • Start from accelerators. Industry accelerators (Healthcare, Financial Services, Manufacturing) contain production-grade assets that would take weeks to build from scratch.
  • Publish back to Exchange. If you build something reusable, publish it to your org's Exchange so your team can find and extend it.

🚀 Deployment

  • Start in Sandbox, promote to Production. Always deploy to Sandbox first. Verify it works, then redeploy to Production.
  • Use Micro workers for development. 0.1 vCore (Micro) workers are sufficient for development and testing. Scale up for production.
  • Version your API specs. Before publishing to Exchange, make sure your spec has a proper semantic version (1.0.0, 1.1.0, etc.).

🤖 Working with Claude Code

  • Be specific with prompts. "Add an HTTP listener on port 8081 with path /api/v1/patients" works better than "add a listener."
  • Chain operations naturally. Claude understands multi-step requests: "Create a project, add a flow with an HTTP listener and a database connector, then deploy to Sandbox."
  • Ask Claude to explain. If you're unfamiliar with a concept, ask: "Explain what the MCP Connector does" or "What scopes do I need for API Manager?"
  • Review generated flows. Always review the generated Mule XML before deploying. Claude is powerful but the output should be verified, especially for production deployments.

🛠 Troubleshooting

  • Check connection first. If something isn't working, run claude mcp list to verify the server is connected.
  • Re-add if stuck. If you hit persistent errors, claude mcp remove mulesoft then re-add. This clears any cached state.
  • Check vCore availability. Deployments fail silently if your org has no available vCores. Check Runtime Manager > Usage to verify.
